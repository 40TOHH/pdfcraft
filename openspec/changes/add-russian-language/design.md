# Дизайн интеграции русского языка

## Архитектурные решения

### Обновление конфигурации локалей
Для добавления русского языка в приложение, необходимо обновить файл `src/lib/i18n/config.ts`, добавив 'ru' в массив `locales` и соответствующую информацию в объект `localeConfig`.

### Интеграция с системой маршрутизации
Система маршрутизации next-intl автоматически обработает новую локаль, поскольку она использует конфигурацию из `src/lib/i18n/config.ts`.

### Обновление компонента выбора языка
Компонент выбора языка будет автоматически включать русский язык после обновления конфигурации, поскольку он использует список локалей из центральной конфигурации.

## Компромиссы

### Производительность
Добавление нового языка увеличивает размер сборки незначительно, поскольку переводы загружаются по требованию (lazy loading).

### Поддержка
Добавление нового языка увеличивает объем необходимых переводов при добавлении новых функций в интерфейс.

## Альтернативы

### Отложенная загрузка переводов
Вместо включения всех переводов в основную сборку, можно реализовать отложенную загрузку переводов по мере необходимости. Однако текущая архитектура next-intl уже эффективно обрабатывает загрузку переводов, поэтому дополнительная оптимизация не требуется на данном этапе.

### Использование стороннего сервиса локализации
Можно было бы использовать сторонний сервис для управления переводами, но это усложнило бы архитектуру и увеличило бы зависимости. Встроенная система next-intl уже хорошо справляется с задачей.
